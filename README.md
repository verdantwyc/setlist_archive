# 一川草色・典藏

一川草色・典藏是一个多语言节目单彙整网站，用来整理音乐现场、活动场次、演出曲目、创作者资料与歌曲平台连结。

网站以「资料库」为核心：维护 `artists`、`tracks`、系列/场次索引与各场次 `setlists` 后，页面会自动组合出首页、主题页、场次节目单、曲目卡与创作者卡。内容支持简体中文、繁体中文与英文。

## 网站内容

- 演出系列索引：以主题或系列彙整场次。
- 场次节目单：每个场次独立记录正场曲目与安可曲。
- 曲目库：收录曲名、演唱者、词曲编、专辑说明与平台连结。
- 创作者库：收录创作者多语言名称、别名与简介。
- 多语言切换：同一份资料可在简体、繁体、英文之间切换显示。
- 自动关联：节目单只需写简体曲名与创作者名，页面会从曲目库与创作者库补齐多语言名称、原唱/原作资讯与平台连结。

## 资料结构

```text
index.html
main.js
data/
  museum.json
  series.json
  tracks.json
  artists.json
  setlists/
    {venueId}.json
```

`index.html` 不内嵌资料。正式显示内容全部来自 `data/` 目录下的 JSON 档案，因此更新资料时只需要维护 JSON。

## 资料维护原则

- `data/artists.json` 是创作者资料库，以 `zh_cn` 简体名称作为主要识别键。
- `data/tracks.json` 是曲目资料库，以 `zh_cn` 简体曲名作为主要识别键。
- `data/series.json` 是系列与场次索引，决定首页与系列页出现哪些主题和场次。
- `data/setlists/{venueId}.json` 是单一场次节目单，档名需对应 `data/series.json` 内的场次 `id`。
- 场次与系列的曲目数量由页面自动计算，不需要手动填写。

## 适合用途

这个项目适合用来建立公开的演出资料档案、歌单资料库、音乐会纪念页、粉丝向节目单彙整站，或任何需要长期维护多语言曲目资料的静态网站。

## 备注

本网站是静态页面，不需要后端服务。只要 JSON 格式正确、档案路径保持一致，部署后即可由浏览器直接读取资料并显示。
